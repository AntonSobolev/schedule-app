Проект — это ретро‑терминальное веб‑приложение «Распорядок дня» для ручного планирования и отслеживания дня с дискретностью 30 минут. Визуально и концептуально приложение имитирует текстовый интерфейс 80‑х: моноширинный шрифт, псевдографика, символы вместо виджетов.

Цель: дать пользователю максимально простую и наглядную модель дня, где он сам «рисует» свой график и затем проживает его в реальном времени.

Базовые принципы

Один слот = 30 минут = один символ

День фиксирован: 08:00 – 21:00 (13 часов = 26 слотов)

Никаких автопланов, алгоритмов, приоритетов — только ручной ввод

Всё выровнено по символам. Любое расхождение ломает интерфейс

В каждый момент времени мигает только один текущий слот

Символьная модель

Используются строго следующие символы:

Символ  Значение
▢ пустой будущий слот
▣ текущие 30 минут (мигает)
■ прошедший рабочий слот
▨ слот отдыха

Никаких других вариантов или замен.

Структура интерфейса (4 секции)

Экран логически разбит на 4 зоны, все строки одной высоты и шрифта.

┌ ЛЕВЫЙ ВЕРХ ────────┬ ПРАВЫЙ ВЕРХ ────────────────┐
│ текст              │ визуальная шкала времени    │
├────────────────────┼─────────────────────────────┤
│ ЛЕВЫЙ НИЗ          │ ПРАВЫЙ НИЗ                   │
│ названия задач     │ слоты задач                 │
└────────────────────┴─────────────────────────────┘
1. Левый верх

Текстовая информация:

Диапазон дня: 08:00.21:00

Текущее время в формате [HH:M] (час + номер получаса)

Легенда символов: ▢.▣.■.▨

Пользователь ничего здесь не вводит.

2. Правый верх

Главная временная шкала дня — агрегированное представление всех задач.

Содержит ровно 26 символов‑слотов

Над ней стоят метки часов:

.9a      .12      .3p      .6p      .9p

Метки и точки должны строго совпадать по позициям со слотами

Показывает:

▢ будущие слоты

■ прошедшие

▨ если в любой задаче в этом получасе стоит отдых

▣ только для текущего получаса

3. Левый низ — список задач

Режим «блокнота».

Каждая строка — отдельная задача

Пользователь печатает произвольный текст

Допускаются чекбоксы внутри текста:

▢Task

клик → ☒Task + зачёркивание

Важно:

Это не список задач, а просто текст

Чекбоксы — декоративная/психологическая функция

4. Правый низ — сетка слотов задач

Для каждой строки задачи:

Справа отображаются 26 слотов

Они визуально синхронизированы с верхней шкалой

Разделитель | у всех строк должен стоять строго в одной колонке

Режимы работы
1. Режим редактирования (по умолчанию)

Пользователь создаёт расписание.

Управление (клавиатура):

Печатать — ввод названия задачи

Tab — переключение между названием и слотами

←↑↓→ — навигация по слотам

Space — поставить/убрать рабочий слот (▢)

R — поставить/убрать слот отдыха (▨)

Enter — новая задача

F5 — запуск дня

Управление (мышь):

ЛКМ по слоту — рабочий слот

ПКМ по слоту — отдых

Контекстное меню запрещено

2. Режим выполнения (run)

После F5:

Редактирование запрещено

Время идёт автоматически

Каждые 30 минут:

предыдущий слот → ■

следующий → ▣ (и мигает)

Дополнительно:

В правом нижнем отображается прогресс по текущей задаче

В конце дня приложение издаёт звук

Выход: Q

Временная логика

День всегда начинается в 08:00

Текущий слот вычисляется как:

(текущее_время − 08:00) / 30 минут

Если текущее время вне диапазона — слот = -1

Никогда не мигают два слота одновременно

Критические требования (часто ломались)

Один слот = один символ (не два)

Все шрифты: один размер, один вес, один family

Верхняя шкала должна учитывать все задачи, включая отдых

Разделитель | никогда не сдвигается

Метки .9a .12 .3p .6p .9p выравниваются по символам, не «на глаз»
